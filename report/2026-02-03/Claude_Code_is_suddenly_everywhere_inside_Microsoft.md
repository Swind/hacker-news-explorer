# 2026-02-03: Claude Code 突然在微軟內部無處不在

**來源：** Hacker News
**故事 ID：** 46854999
**Hacker News 連結：** https://news.ycombinator.com/item?id=46854999
**網址：** https://www.theverge.com/tech/865689/microsoft-claude-code-anthropic-partnership-notepad
**分數：** 356 | **評論數：** 474

## 摘要

微軟正在內部大規模推廣使用 Anthropic 的 Claude Code，即使該公司同時向客戶銷售競爭產品 GitHub Copilot。根據報導，微軟已要求 CoreAI 團隊、Experiences + Devices 部門（負責 Windows、Microsoft 365、Outlook、Teams 等）的數千名員工安裝並使用 Claude Code，甚至鼓勵非技術人員使用它來進行程式設計和原型開發。

## 為什麼有趣

這是一個**高度爭議性且具重大商業意義**的事件，原因如下：

### 1. **戰略矛盾的揭露**
- 微軟是 OpenAI 的主要投資者（約 130 億美元），卻在內部大力採用競爭對手 Anthropic 的產品
- 微軟向客戶銷售 GitHub Copilot，但內部卻更青睞 Claude Code
- 這暴露了微軟 AI 產品線內部的競爭和矛盾

### 2. **重大合作關係變化**
- 微軟現已成為 Anthropic 的頂級客戶之一
- 兩家公司在 11 月簽署協議，允許 Microsoft Foundry 客戶使用 Claude 模型
- Anthropic 承諾購買 300 億美元的 Azure 運算資源
- 微軟銷售人員現在可以將 Anthropic 模型納入 Azure 銷售配額——這對非 OpenAI 模型來說極不尋常

### 3. **技術優勢的認可**
- Claude Code 在易用性方面被認為優於 GitHub Copilot
- 微軟正在讓非技術人員（設計師、專案經理）使用 AI 來撰寫程式碼
- 這標誌著 AI 程式碼生成從專業工具向更廣泛用戶群的擴展

### 4. **對行業的深遠影響**
- 可能進一步壓縮初級開發人員的就業機會
- 推動「AI 代理」更自主地創建程式碼的趨勢
- 挑戰傳統軟體工程工作流程

## 主要討論點

### **微軟產品命名的混亂**

評論區的焦點之一是微軟長期以來的產品命名問題：

> "Microsoft Copilot"、"GitHub Copilot"、"Microsoft 365 Copilot"、"Copilot CLI"——到處都是 Copilot，但沒人清楚哪個是哪個。

許多用戶分享了微軟產品命名的混亂歷史：
- 現在的「Copilot」狂潮
- 之前的「365」、「One」、「Live」
- 20 年前的「.NET」（所有東西都叫 .NET）
- 90 年代的「Active」系列

### **OpenAI 合作關係的質疑**

評論者指出微軟這一舉動的深層含義：

> "現在很明顯，他們甚至沒有在使用自己的 LLM 產品，那為什麼還要有人為 Copilot 付費？"

有人認為這暴露了微軟 CEO Satya Nadella 在 AI 戰略上的重大失誤：

> "Nadella 從頭到尾搞砸了 AI...如果他不儘快糾正這些錯誤，他將失去微軟的大部分進展。"

### **AI 代理和未來工作模式的討論**

關於 AI 讓非技術人員寫程式的影響：

- **擔憂方**：這將進一步威脅初級開發者的工作機會
- **樂觀方**：讓更多人能夠快速實現想法，提高生產力
- **中立方**：這只是工具的演進，需要適應新的工作方式

### **微軟的帝國建造問題**

一些評論分析了大型科技公司內部重複產生的原因：

> "這經常發生，因為內部的人被激勵去建立自己的帝國。如果有人想要升職或成為高管，在現有的傘下工作有天花板...所以他們建立新的東西。"

### **Microsoft vs. Anthropic vs. OpenAI 三方關係**

評論者對這三方關係的複雜性感到困惑：
- 微軟投資 OpenAI，但在內部使用 Anthropic
- Anthropic 向微軟購買運算資源
- 微軟可能未來會向雲端客戶銷售 Claude Code

這形成了一個複雜的競爭合作網絡。

## 評價

**⚠️ 高度爭議且值得關注**

這個故事之所以重要，是因為它揭示了科技行業正在發生的深層變化：

1. **AI 工具戰爭進入新階段**：不再是 OpenAI vs Google，而是 OpenAI、Anthropic、微軟之間更複雜的競爭合作關係

2. **大型科技公司的戰略困境**：微軟既要保護其 OpenAI 投資，又要認可 Anthropic 的技術優勢，暴露了大公司在快速變化的 AI 市場中的兩難

3. **軟體開發的民主化 vs. 專業化**：非技術人員使用 AI 寫程式，可能根本性地改變軟體開發行業

4. **商業模式的不確定性**：如果微軟自己都不相信 GitHub Copilot 是最好的，客戶為什麼要買？

**推薦閱讀評論**：這個討論串不僅揭示了微軟的產品策略問題，還反映了整個行業對 AI 未來的深刻思考和焦慮。
