# 2026-02-04: Anki ownership transferred to AnkiHub

**來源：** Hacker News
**故事 ID：** 46861313
**Hacker News 連結：** https://news.ycombinator.com/item?id=46861313
**網址：** https://forums.ankiweb.net/t/ankis-growing-up/68610
**分數：** 547 | **評論數：** 234

## 摘要

Anki 的創始人 Damien Elmes (dae) 將 Anki 的領導權移交給 AnkiHub 團隊。AnkiHub 是由 Nick (The AnKing，住院醫師) 和 Andrew Sanchez (研究軟體工程師) 創立的教育科技公司。雙方承諾 Anki 將維持開源，保持現有定價模式，並強調沒有外部投資者參與。

## 為什麼重要/爭議性

這是一個**高度爭議性**的重大事件：

1. **影響範圍廣泛**：Anki 是全球數百萬學習者依賴的間隔重複學習工具
2. **社群信任問題**：長期由個人維護的開源專案轉向公司運營，引發對「enshittification」(品質惡化) 的擔憂
3. **商業化疑慮**：AnkiHub 本身是付費服務，社群擔心未來整合可能影響 Anki 的開源性質
4. **治理結構未明**：儘管承諾透明化，但具體的治理模型和決策流程仍在「探索中」

## 主要討論點

### 支持與期待
- **解決 bus factor 問題**：Anki 長期由單一維護者 Damien 主導，轉移權限可分散風險
- **改善 UI/UX**：Anki 的使用者界面長期被批評過時，專業設計資源的加入是正面進展
- **加速開發**：更多工程師和設計師參與可解決長期積壓的功能請求
- **跨平台統一**：AnkiDroid 核心貢獻者 David Allison 將加入，有望改善生態系碎片化問題

### 擔憂與質疑
- **AnkiHub 的商業模式**：AnkiHub 已有付費訂閱制，社群擔心未來可能將 Anki 核心功能付費化
- **開源承諾的可執行性**：雖然承諾維持開源，但缺乏法律層面的保障機制
- **決策透明度**：許多關鍵問題（治理結構、決策流程）仍在「探索中」，沒有明確答案
- **AnkiDroid 的獨立性**：儘管聲稱 AnkiDroid 保持獨立，但核心維護者加入 AnkiHub 引發利益衝突擔憂

### 技術與使用者影響
- **add-on 生態系**：承諾提供更穩定的 API 和更好的文件，對開發者是正面消息
- **移動端應用**：iOS 版本 $25 的定價長期受爭議，社群期待價格調整或統一免費
- **資料遷移與相容性**：使用者擔心過渡期間的資料安全和跨平台同步問題

## AnkiHub 是什麼？

根據官方說明：
- **創立者**：Nick (The AnKing，知名醫學 Anki 牌組作者) 和 Andrew Sanchez (研究軟體工程師)
- **起源**：從醫學教育社群成長起來，最初是協作製作 Anki 牌組的平台（如 AnKing Step Deck）
- **現狀**：提供協作功能、共享牌組同步、LLM 功能的付費服務
- **資金來源**：聲稱完全自資（self-funded），無外部投資者

## 承諾與保證

AnkiHub 團隊在公告中做出的承諾：
- ✓ Anki 核心程式碼維持開源
- ✓ 無外部投資者參與
- ✓ 保持現有定價模式
- ✓ 拒絕操縱性設計模式
- ✓ 尊重使用者代理權
- ✓ 改善透明度和決決策流程

## 未來觀察重點

1. **治理模型建立**：官方承諾在數週內制定更正式的治理結構，需觀察是否具備社群監督機制
2. **Anki 與 AnkiHub 的整合邊界**：兩者產品線如何區分，是否會出現功能綁定
3. **開源社群參與度**：志願貢獻者的角色是否被弱化，決策流程是否開放
4. **技術債務處理**：UI/UX 改善是否會犧牲進階使用者的自訂能力

## 評價

**高度關注，但需保持審慎樂觀**

這個事件的重要性在於它反映了開源軟體長期可持續性經典問題：當創始者無法或不想繼續主導時，如何確保專案不被「收割」或惡質化。

**正面因素**：
- AnkiHub 團隊深諳 Anki 生態，是長期使用者而非外部資本
- 明確承諾維持開源和無投資者干預
- 有經驗豐富的開發者（David Allison）加入

**風險因素**：
- 缺乏法律層面的開源承諾保障
- 商業公司與開源專案的利益衝突潛力
- 過渡期的許多「未知數」

**建議**：使用者社群應密切監督過渡進程，利用開源特性確保分支（fork）選項始終可行。AnkiHub 需要快速建立具體的治理透明度來贏得信任。
