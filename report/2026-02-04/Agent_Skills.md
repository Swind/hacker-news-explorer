## 摘要

Agent Skills 是一個標準化格式，旨在為 AI Agents 提供可重複使用的程序知識和企業特定情境。它允許開發者將專業領域知識、工作流程和功能打包成可跨不同 Agent 產品使用的標準化「技能」模組。

## 為什麼有趣

這是一個**技術意義與爭議性並存**的話題：

### 技術意義
1. **解決 Context Window 限制**：透過 index 機制讓 AI 只在需要時載入特定技能，避免將所有文件一次性塞入 context
2. **跨平台互操作性**：標準化格式讓同一個技能可以在不同的 Agent 產品中重複使用
3. **實證效能提升**：有實驗顯示使用 skills 可以讓 Qwen3-0.6B 在 HumanEval 上提升 +6 分，甚至首次執行就超越基準分數
4. **企業知識管理**：提供一個標準化的方式來捕獲和管理組織知識

### 爭議性
社群對此標準化存在明顯分歧意見，這正是它有趣的地方。

## 主要討論點

### 1. 支持觀點

**效能提升**
- 經實驗證明，skills 能顯著提升小型模型在程式設計任務上的表現
- 透過提供 domain-specific knowledge，讓模型在專業任務上表現更好

**發現機制改善**
- 自動生成 skills index，讓 Agent 知道有哪些可用技能
- 避免用戶需要手動告訴 Agent 去讀特定文件

**標準化與互操作性**
- 不同 Agent 平台可以使用相同的技能套件
- 一次編寫，多處部署

### 2. 批評觀點

**「Bitter Lesson」質疑**
- 許多人認為這只是將舊範式的實踐映射到新範式
- 質疑標準化格式（skills.md、特定標題、段落長度）是否有必要
- 認為只要提供清晰、結構化的文件即可，不需要特殊格式

**現實問題**
- 目前 Agent 仍然經常「忘記」使用 skills，用戶仍需手動提醒
- Skills 描述若不夠精確，Agent 可能選擇錯誤的技能
- 發現機制仍然不夠智能

**與現有文件的差異**
- 批評者認為這與傳統良好的文件實踐差異不大
- 質疑是否需要新的標準，還是只需更好的 README 結構

### 3. 中立/技術討論

**Context 管理策略**
- 討論不同的文件組織方式：
  - 單一 AGENTS.md 文件
  - README index 連結到其他文件
  - Skills with progressive disclosure
- Token 效率與任務成功率的權衡

**「Anthropomorphism」反思**
- 有人認為整個 Agent 概念本身就是一種認知偏差
- 我們試圖用「經理」、「團隊」、「專業化」等人類概念來理解 AI
- 實際上可能更簡單：將任務饋送到單一引擎即可

**與 OpenAPI/Swagger 的比較**
- 有人將 skills index 與 API 文件標準做比較
- 質疑是否真的需要新的標準

## 技術解析

### Agent Skills 如何運作

1. **Index 機制**：系統從所有技能的描述中構建一個 index
2. **Context 注入**：index 在每次對話開始時被傳入 LLM
3. **按需載入**：LLM 讀取技能描述後決定是否載入完整技能內容
4. **Token 節省**：Claude 被設定在讀完描述後停止，只在有用時才消耗更多 token

### 與傳統文件的差異

| 特性 | 傳統文件 | Agent Skills |
|------|----------|--------------|
| 結構 | 自由格式 | 標準化標題、段落長度 |
| 發現 | 人工查找 | 自動 index |
| 載入 | 全量或手動 | 按需 progressive disclosure |
| 互操作性 | 平台特定 | 跨平台標準 |

## 社群共識

1. **文件組織確實重要**：無論是否採用 skills 標準，良好的文件結構和 modular approach 都有幫助
2. **標準化有爭議**：是否有必要建立全新標準，還是應該改進現有文件實踐
3. **現實效能**：技能確實能提升某些情況下的表現，但不是銀彈
4. **長期價值**：質疑這是否只是一個過渡期的 hack，未來會被更好的模型能力取代
