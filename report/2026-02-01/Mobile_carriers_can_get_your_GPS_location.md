# 移动运营商可以获取你的GPS定位

## 文章概述

这篇文章揭示了一个令人震惊的隐私事实：移动运营商可以通过蜂窝网络协议（RRLP和LPP）直接从用户的设备获取精确的GNSS（GPS、GLONASS、Galileo、北斗）坐标，精度可达**米级别**。

### 关键技术细节

1. **2G/3G协议（RRLP）**: Radio Resources LCS Protocol
2. **4G/5G协议（LPP）**: LTE Positioning Protocol
3. **工作原理**: 这些是控制平面定位协议，运营商只需向设备发送请求，设备就会"默默地"回复其精确的GPS坐标
4. **不可见性**: 这些协议在蜂窝网络内部运作，对终端用户几乎完全不可见

### 重要背景

- Apple在iOS 26.3中引入了新隐私功能，限制通过基站获取"精确定位"数据
- 该功能仅适用于使用2025年Apple自研基带的设备
- 文章指出这只是一个开始，但用户仍应能完全禁用GPS位置响应

## 为什么这很重要

### 隐私影响

1. **被动计算变为主动上报**: GNSS本身是被动技术（像看路标），但这些协议让设备主动上报位置
2. **法律漏洞**: DEA在2006年使用法院命令（而非搜查令）获取GPS坐标
3. **大规模监控**: 以色列Shin Bet通过运营商数据中心持续追踪所有人的位置
4. **COVID-19追踪**: 以色列在2020年3月仅用几周就启动了基于运营商数据的接触者追踪系统，证明数据精度远超基站三角定位

### 技术意义

1. **运营商控制层面的权力**: 这些协议是蜂窝标准内置的，无法通过常规手段检测或阻止
2. **跨代际问题**: 从2G到5G都存在类似机制
3. **国际安全风险**: 可能被外国运营商利用（如沙特阿拉伯滥用SS7协议）
4. **行业文化问题**: 电信行业的诚信度和管理能力堪忧

### 协议架构

- **控制平面协议**: 与用户数据分离，难以监控
- **请求-响应模式**: 运营商主动请求，设备被动响应
- **标准化内置**: 这是3GPP标准的一部分，不是后门或漏洞

### 技术深度分析

#### RRLP/LPP协议细节

**工作流程**:
1. 运营商向设备发送定位请求
2. 设备检查是否启用GNSS
3. 如果启用，设备计算位置并返回坐标
4. 整个过程无需用户交互或确认

**精度对比**:
- 基站三角定位: 50-500米（市区）到数千米（郊区）
- GNSS协议获取: 3-10米（与地图应用相同精度）

**关键差异**:
- GNSS是被动计算（像看路标）
- RRLP/LPP让设备主动上报位置（像向陌生人报告你看到路标）

#### Apple的新功能

**iOS 26.3改进**:
- 限制运营商获取"精确位置"
- 仅适用于2025年Apple自研基带设备
- 可能返回模糊化的基站定位

**不足之处**:
- 仅针对新设备，旧iPhone用户无保护
- 未完全禁用，只是限制精度
- 无用户通知机制
- Android无类似保护
