# 2026-02-05: Notepad++ 供應鏈攻擊分析

## 摘要

Kaspersky 發布關於 Notepad++ 供應鏈攻擊的詳細分析，揭露了從 2025 年 6 月至 12 月期間發生的複雜攻擊鏈。攻擊者透過入侵 Notepad++ 的更新基礎設施（WinGUp updater），在四個月內使用了三種不同的感染鏈和載荷，目標針對越南、薩爾瓦多、澳大利亞的個人用戶，以及菲律賓政府機構、薩爾瓦多金融組織和越南 IT 服務提供商。儘管攻擊手法多變，Kaspersky 的解決方案成功攔截了所有識別出的攻擊。

## 為什麼有趣

這是一個**技術價值極高**的安全案例研究：

1. **真實的供應鏈攻擊案例**：展示攻擊者如何透過入侵更新機制來獲得信任執行
2. **多變的攻擊手法**：四個月內使用三種不同的感染鏈，顯示攻擊者的適應性
3. **精密的技術細節**：包含完整的 IoC（入侵指標）、SHA1 雜湊值和 C2 伺服器位址
4. **目標性攻擊**：針對特定組織和地區的精密攻擊，非大規模撒網
5. **繞過防禦**：使用合法軟體（ProShow）漏洞而非常見的 DLL sideloading 技術
6. **持續存取**：攻擊者在託管提供商事件後仍能維持對內部服務的存取長達數月

## 主要討論點

### 更新機制的脆弱性
- **高價值目標**：更新機制是完美的攻擊目標，因為用戶信任更新來源
- **檢測困難**：惡意程式碼以完整用戶權限從「信任」來源執行，大多數端點保護不會標記合法發布者的更新基礎設施
- **六個月的窗口期**：供應鏈攻擊難以檢測，造成長時間的暴露窗口

### Windows 生態系統的包裝問題
- **缺乏統一包裝管理器**：Windows 長期以來缺乏像 Linux 那樣的完善包裝系統
- **不安全的下載來源**：許多程式（包括 FOSS）從充滿廣告的可疑網站下載，每個應用程式以不同方式自我更新
- **Microsoft Store 的失敗**：本可以解決此問題，但設計不佳且未廣泛採用
- **WinGUp 的簡陋**：Notepad++ 的更新機制只是共享託管帳戶上的 PHP 腳本指向可執行檔的 URL

### Linux 與 Windows 的安全對比
- **Debian/RPM 的記錄**：至今未有人成功入侵 Debian 的 APT 儲存庫或 Red Hat 的 RPM 儲存庫
- **更多資源和眼睛**：擁有更多資源的組織可以做得比單一 PHP 腳本更好
- **信任問題**：有使用者表示不信任 Microsoft，因此使用瀏覽器安裝軟體

### 沙盒與權限管理
- **iOS 風格的未來**：討論傾向於沙盒化的應用程式，而非完全自由的桌面應用程式
- **權限平衡**：安全性和可用性之間的平衡，過度安全會導致軟體難用
- **能力型安全**：提議實作適當的能力型安全（capability-based security）

### 防禦建議
- **分階段部署**：組織應該實施分階段推出和網路監控
- **監控異常連線**：監控常見應用程式的意外外傳連線
- **密碼學驗證**：包裝管理器至少增加了密碼學驗證這道屏障
- **網路連線預設關閉**：有使用者指出他們會關閉 Notepad++ 的網路連線功能
