# 2026-02-02: Microsoft 縮減 Windows 11 AI 功能，策略重大轉向

**來源：** Hacker News
**故事 ID：** 46854951
**Hacker News 連結：** https://news.ycombinator.com/item?id=46854951
**網址：** https://www.windowscentral.com/microsoft/windows-11/microsoft-is-reevaluating-its-ai-efforts-on-windows-11-plans-to-reduce-copilot-integrations-and-evolve-recall
**分數：** 473 | **評論數：** 79

## 摘要

Microsoft 正在重新評估其在 Windows 11 的 AI 策略，計劃縮減或移除某些 Copilot 整合功能，並重新設計备受爭議的 Recall 功能。這一轉變反映了用戶對 Windows 11 過度強推 AI 功能的強烈反彈，標誌著 Microsoft 從「AI 第一」戰略向更務實方向的調整。

## 為什麼有趣

這是一個**值得關注的業界轉折點**，具有多重意義：

1. **用戶力量的勝利**：科技巨頭第一次因為用戶強烈反彈而大幅回滾 AI 推廣策略
2. **產業趨勢指標**：顯示「AI 所有東西」的熱潮正在冷卻，業界開始理性思考 AI 的適當應用場景
3. **產品治理失敗案例**：展現了大型科技公司在 hype 驅動下如何失去產品品味和用戶中心思维
4. **品牌價值損害**：Microsoft 甚至不惜犧牲 Office 這個全球知名品牌來推廣 Copilot，造成巨大反效果

## 主要討論點

### 1. 治理與品味失敗

HN 社群普遍認為這是**治理和產品品味的完全失敗**：

> "顯然這是治理的完全失敗。他們首先應該考慮的是這些功能是否合理，而不是現在需要'回滾'本就不該構建的功能。"
> 
> "品味來自於知道什麼不該構建。"

核心問題：Microsoft 在 AI 熱潮中失去了判斷力，將 Copilot 按鈕塞進 Notepad、Paint 等根本不需要 AI 的應用程式。

### 2. 激勵結構問題

許多評論指出這是**組織激勵結構失調**的結果：

- PM 和員工需要展示影響力和相關性以獲得晉升
- AI 是當前熱點，參與 AI 項目意味著職業發展
- 即使是愚蠢的主意（如"在 bootloader 中加入 AI"）也被執行
- 「保持某處著火才能保持聚光燈」的病態文化

### 3. 品牌策略災難

Microsoft 將 Office 重新品牌為 "Microsoft 365 Copilot" 的決策被廣泛批評：

> "他們甚至燒毀了世界上最有名的品牌之一——MS Office，只為了試圖洗白他們的 Copilot 名字。做出這個決策的行銷人員緊急需要另謀高就。"

### 4. 用戶流失與 Linux 機遇

多位評論者表示已因此轉向 Linux：

> "就在 Windows 10 即將退役之時，你們發動了我見過最瘋狂的反行銷活動，成功誘使我將日常驅動系統切換到 Linux。"
> 
> "在我有生之年一直在使用 Windows，直到今年。"

這顯示 Microsoft 的策略正在**推走其最忠實的技術用戶群體**。

### 5. 技術基礎 vs 用戶體驗

重要觀察：Windows 的問題不在於內核（NT kernel 仍然穩健），而在於**用戶空間的瘋狂**：

- 控制面板被「愚蠢化」為 Settings
- 強制推廣 Edge、OneDrive、Microsoft 帳號
- 不断改變的 UI 和設置選項
- 不尊重用戶選擇的強制行為

## 技術評估

### Recall 功能的失敗

Windows Recall 功能原本計劃記錄用戶在 PC 上的所有活動，但面臨巨大的安全和隱私反彈：

- 2024 年宣布後引發用戶強烈反對
- Microsoft 被迫推遲發布整整一年
- 據報導，Microsoft 承認 Recall 的當前實現已經失敗
- 正在探索演進該概念的方法，可能會放棄 Recall 名稱

### Copilot 整合審查

目前處於審查中的功能：
- Notepad 中的 Copilot 整合
- Paint 中的 Copilot 整合
- File Explorer 中的 Copilot 按鈕
- 其他內建應用程式的 AI 整合

Microsoft 已暫停為內建應用程式添加更多 Copilot 按鈕的工作。

### 繼續推進的 AI 項目

並非所有 AI 工作都被縮減，以下仍在繼續：
- Semantic Search（語義搜索）
- Agentic Workspace（代理工作空間）
- Windows ML
- Windows AI APIs

這些被視為「底層」AI 能力，對開發者和用戶仍有價值。

## 評價

**值得關注 / 有趣**

這個故事具有重要的歷史意義和現實啟示：

1. **時代轉折標誌**：標誌著「AI 熱潮」從狂熱回歸理性的轉折點
2. **用戶力量的範例**：證明了社群反饋仍然能夠影響科技巨頭的決策
3. **產業警示案例**：所有公司都應該從 Microsoft 的錯誤中學習教訓
4. **未來發展觀察點**：Microsoft 是否能真正做到「修復」Windows 11，還是只是暫時退卻

**關鍵啟示**：技術必須服務於用戶需求，而不是為了滿足公司 KPI 或追逐熱潮而強行植入。當用戶明確表達不滿時，即使是最大的科技公司也必須回應。

**Microsoft 面臨的挑戰**：
- 如何重新贏得技術用戶群的信任
- 如何在推進 AI 和保持產品品味之間找到平衡
- 如何修復 Windows 11 的負面品牌形象
- 如何防止類似的「集體短視」再次發生

對開發者和用戶來說，這是一個觀察大型科技公司如何（以及是否能夠）自我糾正的絕佳案例。
